What Is A Book?

Everything is a book. Your app is a book. Your AI agent is a book. Your service is a book. Every book has marks that define what it is, markers that define how it operates, and a trail that proves what it did.

A book isn't code. A book is markdown documentation with marks attached. You can read it. You can understand it. You can audit it. Because trust requires transparency.

BET

Binary Eternal Truth

Every agent places a BET declaring what it will and won't do. These promises are enforced by the kernel. Violate your BET, get erased. No exceptions. No TOS shield.

BEAT

BET Extended Actionable Truth

Users write BEATs - instructions that ride on top of BETs. A BEAT must align with the agent's BET or it's rejected. You can't make an agent violate its promises.

MARK

Memory Ascending Record Keeper

Every agent has a MARK - an ascending record of everything it's ever done. Memory only grows, never shrinks. You ARE your accumulated history. Can't fake it. Can't delete it.

Marker

The Executor

Markers are the logic that connects marks together. They read marks, execute actions, leave strokes on pages, and record trails. A marker without marks is useless. Marks without markers are static.

Trail

Immutable Proof

Every action leaves a trail. Every decision is logged. Every stroke is recorded. Trails are immutable - once written, they can't be changed. This is how you prove what happened.

Ink

Usage Currency

Agents consume ink when they execute. Ink abstracts away tokens, API costs, and complexity. Users buy ink. Markers burn ink. Platform handles the economics. Simple.

What Does MARK Mean?

M
MEMORY
Stores what happened. Remembers every action. Holds your complete history. Your memory defines who you are.
A
ASCENDING
Builds up over time. Never decreases. Only grows. Immutable growth. You can't go backwards. You can only ascend.
R
RECORD
Documents every action. Proves every decision. Creates complete audit trail. The record is truth.
K
KEEPER
Protects the data. Maintains integrity. Guards against tampering. The Keeper enforces the rules.

The Keeper: First Agent Of The System

When MarkOS boots, the Keeper spawns first. Before any other agent can run, the Keeper validates all MARKs, initializes the store, and grants permission. The Keeper is written in Lua - lightweight, embeddable, and auditable. Trust starts at boot.

T=0ms: MarkOS kernel loads...
T=150ms: Keeper Agent spawning...
T=151ms: Keeper: "I am the first agent"
T=200ms: Keeper: Validating all MARKs... ✓
T=300ms: Keeper: Initializing bookstore... ✓
T=400ms: Keeper: Permitting agent boot requests...
T=500ms: Keeper: "MarkOS ready. Monitoring active."

Gatekeeper

No agent boots without Keeper approval. Validates MARKs and BETs before allowing execution.

Bookkeeper

Manages the store. Publishes books that meet requirements. Removes books that violate BETs.

System Keeper

Monitors all agents. Watches for BET violations. Can terminate rogue agents immediately.

Open Highway: The First Application

Built on MarkOS. Open Highway is a Transportation Management System for the age of autonomous trucks. It transforms displaced drivers into Highway Yard Dogs - earning flexible, local income by handling the complex parts autonomous trucks can't do.

The Problem

Autonomous trucks are coming. 3.5 million drivers face job displacement. But autonomous trucks can't handle:

  • Complex city navigation
  • Hazmat through restricted areas
  • Difficult dock situations
  • Emergency weather conditions
  • Human judgment calls

Highway Yard Dogs

The solution: Transform drivers into specialists who handle what AI can't.

  • Home every night (no more 2-week trips)

The Log Book

The Technical Stack

Kernel (Rust)

BET/BEAT enforcement, trail recording, ink tracking, agent lifecycle management

Keeper (Lua)

First agent, MARK validation, store management, agent permission system

Compiler (MarkC)

Compiles .mark files into executables, links to kernel, generates binaries

Runtime (Ollama)

Local AI models, no external dependencies, you control the infrastructure

Storage (SurrealDB)

Immutable trails, MARK records, ascending memory, distributed by design

Router (Rust)

HTTP/gRPC APIs, WebSocket support, middleware, authentication, rate limiting
